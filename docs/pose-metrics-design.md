# フェーズ2設計：MediaPipe姿勢推定による測定層（v1.12.0候補）

作成日: 2026-07-14　状態: 設計承認待ち
前提文書: `docs/reproducibility-phase0.md`（フェーズ0/1の測定結果と残存課題）

## 1. 目的

1. **判定境界の安定化**: フェーズ1後も残る「run間の評価反転」（例: test1の接地評価6⇔7）を、決定的な実測値でアンカーして解消する
2. **差別化**: ケイデンス・体幹前傾角などの実測値を診断画面に表示する。日本語圏の無料フォーム診断で実測値を出すものは存在しない
3. **診断根拠の裏付け**: 「AIの印象」ではなく「計測値＋AIの解釈」という構造に変え、有料化の信頼基盤にする

スコアの決定権は本フェーズでは引き続きGemini（実測値でアンカー）。完全ルールベース化（スコア＝閾値表の関数）は、閾値の校正データが揃うフェーズ3で判断する。

## 2. アーキテクチャ

```
動画バイト列（app.py:172）
  → 一時ファイル化（tempfile、finallyで削除）
  → [測定層 src/measurement.py]（新規・ローカル実行・Gemini非依存）
      OpenCVでフレーム逐次読み（溜めない）→ MediaPipe PoseLandmarker（liteモデル・リポジトリ同梱）
      → ゲートイベント検出（接地・離地）→ 指標計算 → MeasurementResult
  → 信頼度チェックOK → プロンプトに実測値セクション注入 ＋ 結果画面に実測値カード表示
  → 信頼度チェックNG → 測定スキップ（従来どおりの診断。機能後退なし）
```

グレースフルデグラデーションを原則とする：測定層のいかなる失敗（側面撮影でない・検出不能・複数人・低フレームレート）も診断自体を止めない。

## 3. 計測指標（5つ）

| 指標 | 計算方法 | 頑健性 |
|---|---|---|
| ケイデンス（spm） | 足首y座標の周期（接地イベント間隔） | 高。Garmin実測と照合可能 |
| 体幹前傾角（°） | 肩中点−腰中点ラインと鉛直のなす角の接地時平均 | 高（側面撮影時） |
| 上下動比（%） | 腰中点y振幅 ÷ 身長ピクセル（頭頂−足首） | 中。絶対cmは出さない（単眼のため） |
| オーバーストライド指標（%） | 接地瞬間の足首−腰の水平距離 ÷ 身長ピクセル | 中。接地検出精度に依存 |
| 接地時間比（duty factor %） | 接地フレーム数 ÷ ストライドフレーム数 | 中。30fps以上で有効化 |

各指標に信頼度（検出成功フレーム率・撮影向き判定）を付け、信頼度の低い指標は個別に非表示・非注入とする。

## 4. 入力条件と制約

- 測定が有効なのは「側面から撮影された単独ランナー」の動画。向きはkeypoint配置から自動判定し、非側面はスキップ（スクリーナーの仕様は変えない＝レース動画等の従来ユースケースは診断可能なまま）
- 解析対象は最大15秒相当（長い動画は中央区間をサンプリング）、フレームは長辺640pxに縮小、逐次処理でメモリに全フレームを保持しない
- モデルは pose_landmarker_lite.task（約5.5MB）をリポジトリに同梱（実行時ダウンロードなし）

## 5. プロンプト注入（src/prompts.py）

- `_ANALYZER_BASE_PROMPT` の `{context_section}` 直後に `{measurement_section}` プレースホルダを追加し、既存と同じ `.replace()` 方式で注入（format()禁止の教訓を踏襲）
- 注入文言（骨子）：「以下は本アプリがMediaPipe姿勢推定で計測した実測値です。スコアと診断はこの計測値と整合させてください」＋指標表
- `ANALYZER_SYSTEM_INSTRUCTION` の「外部計測データ言及禁止」ルールに例外を明記：「ただし本プロンプトに含まれるアプリ計測の実測値は診断根拠として使用する」（明記しないとv1.11.0のガードが実測値を拒否しうる）
- 測定スキップ時は `{measurement_section}` を空にし、v1.11.0と完全同一のプロンプトになる（後方互換）

## 6. UI（src/ui/components.py・app.py）

- 診断実行の Step 1 の前に「フォーム計測」の `st.status` を1つ追加（ワーカースレッド内で `st.*` を呼ばない規約を踏襲）
- 結果画面：レーダーチャート直後に実測値カード（計測できた指標のみ表示、「実測値（β）」表記）。ダウンロードMDにも同表を含める
- 測定結果は `st.session_state.last_measurements` に保存（既存の `last_*` パターンに統一）

## 7. 依存関係・デプロイ

- 追加: `mediapipe`（0.10.35。macOS arm64／Linux x86_64ホイールあり・確認済み）、`opencv-python-headless`
- 確認事項（STEP Aで検証）: mediapipeとnumpy 2.4の互換、mediapipe同梱cvとheadless版の共存、Linuxでの libGL 要否（必要なら `packages.txt` 新設）
- Streamlit Cloud（CPUのみ・約2.7GB）でのメモリ・処理時間はSTEP Bで実測

## 9. リスクと対応

| リスク | 対応 |
|---|---|
| mediapipeがStreamlit Cloudで起動しない（libGL/numpy） | STEP Aでローカル検証、STEP Eで本番前にCloud検証。最悪時は測定層を環境フラグで無効化でき、v1.11.0相当で動き続ける |
| 接地検出の精度不足 | 信頼度ゲートで低信頼指標を非表示。ケイデンス・体幹角だけでも価値は成立 |
| 処理時間の増加 | 15秒×縮小フレームで数十秒以内を目標。超える場合はサンプリングをさらに間引く |
| プロンプト変更による再現性の再劣化 | STEP Eでフェーズ0と同一手順のビフォー/アフター測定を必須とする |
